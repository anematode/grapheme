(static) roundDown(x) → {number}

Returns the previous floating point number before x. For example, roundUp(0) returns -Number.MIN_VALUE. See Grapheme.FP.roundUp for implementation explanation. This function should NEVER return -0 or +Infinity, but it can accept those values; roundDown(0) is -Number.MIN_VALUE and roundDown(Infinity) is Number.MAX_VALUE.

(static) roundUp(x) → {number}

Returns the next floating point number after x. For example, roundUp(0) returns Number.MIN_VALUE. Special cases (±inf, NaNs, 0) are handled separately. An interesting special case is -Number.MIN_VALUE, which would normally return -0 and thus must be handled separately. Then, the float is put into a TypedArray, treated as an integer, and incremented, which sets it to the next representable value. roundUp should NEVER return -0 or -Infinity, but it can accept those. On my computer both these functions take about 20 ns / call (October 2020). They need to be performant because they are called so often (every interval function, pretty much).
